Clickable Image Hotspots in Twine

A step-by-step guide to making regions of an image navigable to different passages

This guide walks you through creating clickable areas — called hotspots — on an image in a Harlowe Twine story. Clicking a hotspot takes the player to a different passage, just like a regular link. The technique is more involved than adding a simple text link, but it works reliably across screen sizes and is well suited to interactive maps, room illustrations, character portraits, and other forms of visual navigation.

→

How it works — the big picture

The approach has three main pieces that work together:

Piece 1
A positioned container

Your image sits inside a div with position: relative. This is the anchor that all overlay positions are measured from.

Piece 2
Invisible overlay divs

Transparent div elements are placed on top of the image using JavaScript. Each one covers one hotspot region. Their positions are in percentages, so they scale with the image.

Piece 3
Click navigation

A click event listener on each overlay div calls Harlowe's internal navigation function to move the player to the target passage.

⚠️ Important
The method described in this guide injects CSS into the HTML of your story using JavaScript because it's fairly straightforward to implement, especially as it allows you to quickly use AI to generate the necessary calculations. However, this isn't the only way to create clickable hot spots over images in Twine; for example, you can add the CSS directly in the story's Stylesheet (Story > Stylesheet), or you can use the image map method with HTML and CSS. Feel free to experiment with any of those methods or explore any other ideas you have if this one doesn't meet your needs.

1

Prepare Your Image

Step 1 — Find your image's natural dimensions

Before you do anything else, you need to know the exact pixel width and height of your image at its original (natural) size. Everything else — the hotspot coordinates and the percentage calculations — depends on this number.

  1. Open the image URL directly in a browser tab.

  2. Most browsers display the dimensions in the tab title — for example: forest.jpg (5686 × 3726).

  3. Alternatively, right-click the image on any page → Open image in new tab, then read the dimensions from the tab title.

  4. Write down the width and height. You will need them throughout this guide.

💡 Tip

If your image is hosted on Flickr, the dimensions shown in the tab title when you open the direct image URL are the natural dimensions of that specific file. Use those numbers exactly.

Step 2 — Decide on a display width

You will display the image at a fixed maximum width in your story. Choose a width that fits comfortably in the Twine passage — somewhere between 500px and 800px works well for most screens. You also need the corresponding display height.

To calculate the display height that preserves the aspect ratio: 

/* Formula */
display height = display width × (natural height ÷ natural width)

/* Example: natural size is 5686 × 3726, display width = 2000 */
display height = 2000 × (3726 ÷ 5686) = 1311

Write down your chosen display width and calculated display height.

💡 Tip

You can ask Claude to do this calculation. Tell it your natural dimensions and your chosen display width and ask it to calculate the display height.

2

Map Your Hotspot Regions

Step 3 — Use maschek.hu to draw your regions

You will use a free online tool to draw shapes over the parts of your image you want to be clickable. The tool generates pixel coordinates that you will convert to percentages in the next part.

  1. Go to maschek.hu/imagemap/imgmap in your browser.

  2. Click Image URL and paste in the direct URL of your image, then click OK.

  3. Your image will appear in the editor. Use the shape tools to draw regions:

    • Rectangle — click and drag to draw a box
    • Circle — click the centre, drag out to set the radius
    • Polygon — click to place points, double-click to close the shape

  4. Draw one shape for each hotspot you want. You do not need to fill in the href or alt fields.

  5. When all regions are drawn, scroll down to find the generated code. It will look something like this: 

    <img src="..." usemap="#imgmap123">
<map name="imgmap123">
  <area shape="circle" coords="895,971,587"       href="" />
  <area shape="rect"   coords="1536,380,3080,3012"  href="" />
  <area shape="rect"   coords="3128,712,3616,1412"  href="" />
</map>

  6. Copy this code and paste it somewhere you can refer to it — a text file or a note.

⚠️ Important

Maschek.hu maps at the image's natural size. If your image is very large (e.g. 5686px wide), the coordinates will be large numbers. That is fine — you will convert them to percentages in the next step.

Watch out for tiny accidental shapes — for example coords="1544,360,1568,364" is a rectangle only 24×4 pixels. These are usually stray clicks from the drawing tool. Delete them before continuing.

3

Convert Coordinates to Percentages

The overlay divs use percentage-based positioning so they scale correctly when the image resizes. You need to convert your pixel coordinates into percentages of the image's natural dimensions.

The formulas

For each hotspot region you will calculate four values — left, top, width, and height — all as percentages of the image's natural dimensions.

Value Rectangle — coords are x1,y1,x2,y2 Circle — coords are cx,cy,radius
left % x1 ÷ natural width × 100 (cx − r) ÷ natural width × 100
top % y1 ÷ natural height × 100 (cy − r) ÷ natural height × 100
width % (x2 − x1) ÷ natural width × 100 (r × 2) ÷ natural width × 100
height % (y2 − y1) ÷ natural height × 100 (r × 2) ÷ natural height × 100

Let Claude do the calculations

Rather than calculating by hand, you can ask Claude to generate all the percentage values for you. Use the prompt template below — fill in your own numbers and paste your <map> code where indicated.

📋 Prompt template for Claude

I have an image with natural dimensions [WIDTH] × [HEIGHT] pixels. I am resizing it to [DISPLAY WIDTH] × [DISPLAY HEIGHT] for display in Twine. I mapped hotspot regions using maschek.hu and got these coordinates:

[PASTE YOUR FULL <map> CODE HERE]

Please calculate the left, top, width, and height as percentages for each area so I can use them as CSS overlay divs. Show me the working for each one, flag any tiny or overlapping regions, and give me the final values ready to copy into the JavaScript array.

Replace the bracketed parts with your actual numbers and paste your <map> code in full.

💡 Tip

Claude will show the working step by step, check for tiny accidental regions, flag overlapping shapes, and give you the percentage values formatted and ready to paste into the code template in Part 5.

4

Set Up Twine

Step 4 — Add the navigation code to Story JavaScript

Harlowe's passage navigation function is not directly accessible from a passage script tag. You need to capture a reference to it once at startup and store it in a global variable your passage code can reach.

  1. In Twine, open your story.

  2. In the top toolbar, choose Story, then choose JavaScript.

  3. Paste in exactly this code:

    window.harlowe = window.harlowe || {};
window.harlowe.Engine = Engine;

  4. Close the JavaScript editor and save.

This runs once when the story loads and saves a reference to Harlowe's Engine module where your passage scripts can use it.

⚠️ Important

This relies on an undocumented internal Harlowe feature. It has been stable across all versions of Harlowe 3, but it could theoretically change in a future update. Keep your original coordinate data so you can rebuild the code if needed.

Step 5 — Create your target passages

Before writing the hotspot passage, create the passages each hotspot will navigate to. Give each one a clear, unique name — you will need to type these names exactly in the code.

💡 Tip

Short, simple passage names reduce the chance of typos. Forest, Kitchen, and Bedroom are easier to get right than The Dark Forest Path or Room 2 - Kitchen Area.

5

Write the Passage Code

Step 6 — The full passage template

Create the passage that will display your image with hotspots. Replace all placeholder values with your own image URL, display width, percentage values, and passage names. 

<div id="hotspot-container"
     style="position:relative; display:inline-block; width:100%; max-width:2000px;">
  <img src="YOUR-IMAGE-URL-HERE"
       alt="description of your image"
       style="width:100%; display:block;">
</div>

<script>
(function() {
  var container = document.getElementById("hotspot-container");

  var hotspots = [
    { left:"5.4%",  top:"10.3%", width:"20.7%", height:"31.6%",
      circle:true,  passage:"Passage One"   },
    { left:"27.0%", top:"10.2%", width:"27.2%", height:"70.6%",
      circle:false, passage:"Passage Two"   },
    { left:"55.0%", top:"19.1%", width:"8.6%",  height:"18.8%",
      circle:false, passage:"Passage Three" },
  ];

  hotspots.forEach(function(h) {
    var div = document.createElement("div");
    div.style.position   = "absolute";
    div.style.left       = h.left;
    div.style.top        = h.top;
    div.style.width      = h.width;
    div.style.height     = h.height;
    div.style.cursor     = "pointer";
    div.style.background = "rgba(255,0,0,0.3)"; /* DEBUG — remove when done */
    if (h.circle) div.style.borderRadius = "50%";

    div.addEventListener("click", function() {
      window.harlowe.Engine.goToPassage(h.passage);
    });

    container.appendChild(div);
  });
})();
</script>

What each part does

Line / section What it does
<div id="hotspot-container"> Creates the container the image and overlays sit inside. position:relative is essential — it makes the overlay positions relative to this container, not the whole page.
max-width:2000px Limits how wide the image can grow. Change this number to match your chosen display width.
<img style="width:100%"> Makes the image fill its container. Combined with max-width on the container, this gives a responsive image at a controlled maximum size.
var hotspots = [...] An array — one object per hotspot. Each object contains the four percentage positions, whether it is circular, and the exact name of the passage to navigate to.
rgba(255,0,0,0.3) Red debug tint so you can see where each hotspot sits while building. Change to rgba(0,0,0,0) when you are happy with the placement.
if (h.circle) borderRadius:"50%" Makes the visible debug overlay round for circle hotspots. Has no effect on the click area — which is always the full bounding rectangle — but helps you visualise placement.
window.harlowe.Engine.goToPassage() Calls Harlowe's internal navigation function using the reference you saved in Story JavaScript. This is what actually moves the player to the new passage.
6

Test and Finish

Step 7 — Test with the debug tint on

Before removing the red debug colour, verify that your hotspots are landing in the right places.

  1. Click Play or Test in Twine to run your story.

  2. Navigate to your hotspot passage.

  3. You should see red or pink-tinted regions on top of your image. Each one should cover the area you intended.

  4. If a region is in the wrong place, the percentages need adjusting. Ask Claude to recalculate — give it the original coordinates and the corrected natural dimensions.

  5. Click each hotspot to confirm it navigates to the correct passage.

⚠️ The most common mistake

A passage name mismatch. The string inside passage:"..." in your JavaScript must match the passage name in Twine exactly — including capitalisation and spaces. If clicking a hotspot produces a Harlowe error, check the names first.

Step 8 — Remove the debug tint

Once all hotspots are correctly placed and navigating, make them invisible. Find this line: 

div.style.background = "rgba(255,0,0,0.3)"; /* DEBUG — remove when done */

Change it to:

div.style.background = "rgba(0,0,0,0)";

The hotspots are now invisible but still fully clickable. The cursor will change to a pointer when the player hovers over them, giving a subtle cue that the area is interactive.

Step 9 — Publish to File

  1. In Twine, click your story name → Publish to File.

  2. Save the .html file somewhere you can find it.

  3. Open the exported file in a browser and test it there — not inside Twine's preview — to confirm everything works in the final published form.

  4. If your image is a local file, keep it in the same folder as the .html file, or in an images/ subfolder next to it.

💡 Tip

If you are using a hosted image (Flickr, Unsplash, etc.), the exported .html file is fully self-contained and does not need any extra files alongside it.

✓

Complete Checklist

Use this list to make sure you have not missed anything.

  • Note the natural pixel dimensions of your image (width × height)
  • Decide on a display width and calculate the proportional display height
  • Use maschek.hu to draw hotspot regions and copy the generated <map> code
  • Delete any tiny accidental regions from the generated code
  • Ask Claude to calculate the percentage values for each region (use the prompt template above)
  • Add the two-line Engine capture code to Story JavaScript
  • Create all target passages in Twine with clear, unique names
  • Paste the passage template into your hotspot passage
  • Replace placeholder values: image URL, max-width, hotspot percentages, passage names
  • Test with the red debug tint — confirm positions and navigation
  • Change background to rgba(0,0,0,0) to hide the overlay divs
  • Publish to File and test in a browser

Troubleshooting

Problem Most likely cause and fix
No red boxes visible at all The script tag is not running. Check that your passage is saved and that you are viewing it in Play or Test mode, not just the passage editor preview.
Red boxes appear in completely wrong positions The natural dimensions you used for the calculation do not match the actual image. Re-check the original image size (open its URL directly in a browser tab) and ask Claude to recalculate with the correct numbers.
Clicking a hotspot gives a Harlowe error about goToPassage The Story JavaScript code was not saved, or was typed incorrectly. Re-open Story JavaScript and check that both lines are there exactly as written.
Clicking a hotspot gives a passage-not-found error The passage name in your code does not exactly match the name in Twine. Check capitalisation, spaces, and punctuation in both places.
Hotspots are slightly misaligned after resizing the browser This should not happen with the percentage approach. If it does, check that the container div has display:inline-block and the image has width:100%.
Image does not appear The image URL may be broken or blocked. Try opening the URL directly in a browser. For local images, check that the file is in the same folder as the published .html file.
